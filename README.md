# Marketing Flat-file CMS

This repository acts as a local, headless content management system and automation hub for marketing assets and AI agents. It uses a flat-file approach where the file system itself is the database, storing structured JSON outputs generated by AI (Gemini CLI / Claude Code) which can be reviewed, curated, and analyzed via a lightweight SvelteKit web UI.

## Core Architecture

- **Multi-Tenant Structure:** The system is built to manage multiple clients natively. Each client has its own dedicated workspace (`clients/[tenant]/`), containing their specific configurations (`brand.json`), social media posts, and ad campaigns.
- **AI Content Generators (CLI):** We use CLI agents loaded with specific personas (e.g., `.claude/agents/social-media-copy-creator.md` or `google-ads-creator.md`) to generate content and structures autonomously.
- **Data Storage:** Generated posts, configurations, and media files are stored locally. Textual data and metadata are saved as `.json` files.
- **Web UI (SvelteKit):** A stateless front-end (located in `ui/`) to visually browse clients (`http://localhost:5173/[tenant]`), attach untracked media files, manage draft/approved statuses, and view live API metrics for ad campaigns.

## Features

1. **Social Media Management:** Create, import, edit, and approve JSON-based social media posts. Attach media (images/videos) via the UI.
2. **Google Ads Integration:** 
   - Manage Draft and Approved campaigns locally.
   - Connects to the Google Ads API to fetch **Live Campaign Metrics** (Impressions, Clicks, Cost, Conversions, ROAS, CPA, CTR, Interaction Rate).
   - Generates **Markdown Reports for AI**, exporting historical data and Ad Group breakdowns so agents can suggest optimizations.
3. **Gemini CLI Skills:** Embedded procedural skills (`.skill` files) that teach the CLI agent how to interact with the workspace automatically.

### Roadmap

  - **Scheduling:** Autonomous agents running daily monitoring and weekly reports via Claude Code remote triggers
  - **Approval UI:** Generated content surfaced for one-click approval without touching the terminal
  - **Orchestrator:** A marketing director agent that reads all clients, prioritizes, and delegates to specialized sub-agents

## Gemini CLI Skills

This workspace includes custom skills that transform Gemini CLI into an expert marketing assistant. 

Make sure to install and load them:
```bash
/skills reload
```

### Available Skills:
- **`create-client`**: Teaches the agent the exact folder structure and `brand.json` schema required to onboard a new client into the CMS.
- **`list-google-ads-campaigns`**: Enables the agent to query the SvelteKit backend and list all live campaigns running on Google Ads for a specific client.
- **`get-google-ads-campaign`**: Allows the agent to fetch a deep, analytical report of a specific live campaign (including ROAS, CPA, and Ad Group performance) to provide strategic insights.
- **`deploy-google-ads`**: Deploys an approved Google Ads campaign from a local JSON draft directly to the Google Ads API.
- **`publish-social-post`**: Connects to the Facebook/Instagram Graph API to publish approved social media posts (JSON + image/video) directly to the feed.
- **`optimize-roas`**: Analyzes a bleeding Google Ads campaign and auto-pauses underperforming Ad Groups based on CPA/ROAS thresholds.

## AI Agents

This repository provides specific personas (agents) built to automate different steps of the marketing pipeline:
- **`meta-ads-architect`**: Focuses on outputting perfect, deploy-ready Meta Ads JSON structures (Campaign > Ad Set > Ad) aligned with the client's `brand.json`.
- **`social-media-planner`**: Analyzes the client's persona and tone to generate 30-day content calendars in structured JSON format.
- **`creative-analyst`**: Reads technical Markdown reports generated by the UI and outputs professional, easy-to-understand executive summaries for the business owner.

## Typical Workflow

1. **Generation:** Ask the CLI agent to create content for a specific client (e.g., "Create a Search Network campaign for Acme"). The agent writes `.json` files into `clients/acme/ads/google/` with the status `draft`.
2. **Review:** Open the Svelte UI on your browser at `http://localhost:5173/acme/ads/google`. It will list all drafts.
3. **Curation:** Review the texts, budgets, and ad groups. For social media, attach any necessary images.
4. **Approval & Deployment:** Click "Approve" in the UI. (Future implementation: deployment scripts will push these approved JSONs to Meta/Google APIs).
5. **Analysis:** Use the "Live Campaign Details" page to view interactive charts, or click **"Export Markdown for AI"** to generate a report file to feed back into the agent for optimization.
